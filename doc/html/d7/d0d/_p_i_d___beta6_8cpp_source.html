00057  *      I don't see this function being called all that much (other than from the
00058  *  constructor.)  it needs to be here so we can tell the controller what it's
00059  *  input limits are, and in most cases the 0-1023 default should be fine.  if
00060  *  there's an application where the signal being fed to the controller is

00108  * This function allows the controller's dynamic performance to be adjusted. 
00109  * it's called automatically from the constructor, but tunings can also
00110  * be adjusted on the fly during normal operation
00111  ******************************************************************************/
00112 void PID::SetTunings(double Kc, double TauI, double TauD)
00113 {
00114         //verify that the tunings make sense
00115         if (Kc == 0.0 || TauI < 0.0 || TauD < 0.0) return;
00116 
00117         //we're going to do some funky things to the input numbers so all
00118         //our math works out, but we want to store the numbers intact
00119         //so we can return them to the user when asked.
00120         P_Param = Kc;
00121         I_Param = TauI;
00122         D_Param = TauD;
00123 
00124         //convert Reset Time into Reset Rate, and compensate for Calculation frequency
00125         double tSampleInSec = ((double)tSample / 1000.0);
00126         double tempTauR;
00127         if (TauI == 0.0) 
00128                 tempTauR = 0.0;
00129         else 
00130                 tempTauR = (1.0 / TauI) * tSampleInSec;
00131 
00132         if (inAuto)
00133         {       //if we're in auto, and we just change the tunings, the integral term 
00134                 //will become very, very, confused (trust me.) to achieve "bumpless
00135                 // transfer" we need to rescale the accumulated error.
00136                 if(tempTauR != 0.0) //(avoid divide by 0)
00137                         accError *= (kc * taur)/(Kc * tempTauR);
00138                 else
00139                         accError = 0.0;

00141

00143         taur = tempTauR;
00144         taud = TauD / tSampleInSec;

00199 /* Compute() **********************************************************************
00200  *     This, as they say, is where the magic happens.  this function should be called
00201  *   every time "void loop()" executes.  the function will decide for itself whether a new
00202  *   pid Output needs to be computed
00203  *
00204  *  Some notes for people familiar with the nuts and bolts of PID control:
00205  *  - I used the Ideal form of the PID equation.  mainly because I like IMC
00206  *    tunings.  lock in the I and D, and then just vary P to get more 
00207  *    aggressive or conservative
00208  *
00209  *  - While this controller presented to the outside world as being a Reset Time
00210  *    controller, when the user enters their tunings the I term is converted to
00211  *    Reset Rate.  I did this merely to avoid the div0 error when the user wants
00212  *    to turn Integral action off.
00213  *    
00214  *  - Derivative on Measurement is being used instead of Derivative on Error.  The
00215  *    performance is identical, with one notable exception.  DonE causes a kick in
00216  *    the controller output whenever there's a setpoint change. DonM does not.
00217  *
00218  *  If none of the above made sense to you, and you would like it to, go to:
00219  *  http://www.controlguru.com .  Dr. Cooper was my controls professor, and is
00220  *  gifted at concisely and clearly explaining PID control
00221  *********************************************************************************/
